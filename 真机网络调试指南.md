# 真机网络请求调试指南

## 问题描述
在模拟器上可以正常查询余额,但打包安装到真机后无法查询地址余额。

## 可能的原因

### 1. HTTP vs HTTPS
- **Android 9.0 (API 28) 及以上版本默认禁止明文 HTTP 请求**
- 模拟器可能运行在较低API级别,允许HTTP
- 真机运行在Android 9.0+,默认阻止HTTP请求

### 2. IP地址可达性
- 模拟器和电脑在同一局域网,可以访问局域网IP
- 真机使用移动网络,无法访问局域网IP
- 必须使用公网IP或域名

### 3. 网络超时
- 真机网络环境可能比模拟器慢
- 默认axios超时时间可能不够

## 已实施的解决方案

### 1. 网络安全配置
已创建 `android/app/src/main/res/xml/network_security_config.xml`,允许明文流量:

```xml
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="true">101.35.87.31</domain>
    </domain-config>
</network-security-config>
```

### 2. AndroidManifest配置
已在 `AndroidManifest.xml` 中添加网络安全配置引用:

```xml
<application
    ...
    android:networkSecurityConfig="@xml/network_security_config">
```

### 3. Axios超时配置
在 `blockchainApi.ts` 中配置了30秒超时:

```typescript
const axiosInstance = axios.create({
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
    'Accept': 'application/json',
  },
});
```

## 调试方法

### 查看日志
使用 Android Studio Logcat 或 React Native Debugger 查看日志:

```bash
adb logcat | grep "查询地址 UTXO"
adb logcat | grep "查询地址.*的 UTXO 失败"
adb logcat | grep "响应数据"
```

### 常见错误日志

1. **网络错误**:
   ```
   请求已发送但没有收到响应，可能是网络问题
   ```

2. **证书错误**:
   ```
   CertPathValidatorException
   ```
   → 需要添加网络安全配置

3. **超时错误**:
   ```
   timeout of 30000ms exceeded
   ```
   → 检查网络连接或增加超时时间

4. **地址格式错误**:
   ```
   无效的地址格式: [object Object]
   ```
   → 检查地址数据类型

## 其他注意事项

### 1. 服务器配置
- 确保 `101.35.87.31:18333` 的服务器可以从外网访问
- 检查服务器防火墙规则,开放18333端口
- 验证服务器是否正常运行

### 2. CORS问题
如果API有CORS限制,需要在服务器端配置:
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

### 3. 生产环境建议
生产环境建议使用HTTPS:
- 配置SSL证书
- 使用域名代替IP地址
- 移除明文流量配置
