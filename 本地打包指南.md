## 一、环境准备

### 1.1 检查环境要求

确保本地已安装以下工具：

```bash
# 检查 Node.js 版本（建议 18+）
node --version

# 检查 Java JDK（建议 JDK 17）
java -version

# 检查 Android SDK
adb version

# 检查 Expo CLI
npx expo --version
```

### 1.2 安装 Android SDK（如未安装）

1. 下载并安装 Android Studio: https://developer.android.com/studio
2. 打开 Android Studio → SDK Manager
3. 安装以下组件：
   - Android SDK Platform-Tools
   - Android SDK Build-Tools (建议最新版本)
   - Android 14 (API 34) 或更高版本
4. 配置环境变量：
   - `ANDROID_HOME` = Android SDK 安装路径
   - 将 `%ANDROID_HOME%\platform-tools` 添加到 PATH

## 三、配置签名

### 3.1 创建签名配置文件

在项目根目录创建 `android/keystore.properties` 文件：

```properties
KEYSTORE_FILE=D:/keys/tentech.keystore
KEYSTORE_PASSWORD=tentech
KEY_ALIAS=testalias
KEY_PASSWORD=tentech
```

### 3.2 验证证书文件

```bash
# 检查证书文件是否存在
dir D:/keys/tentech.keystore

# 查看证书信息
keytool -list -v -keystore D:/keys/tentech.keystore -alias testalias
# 输入密码：tentech
```

**预期输出：**
```
Alias name: testalias
Creation date: ...
Entry type: PrivateKeyEntry
Certificate chain length: 1
```

### 3.3 配置 build.gradle（如需要）

确保 `android/app/build.gradle` 包含签名配置：

```gradle
android {
    ...

    // 读取签名配置
    def keystorePropertiesFile = rootProject.file("keystore.properties")
    def keystoreProperties = new Properties()
    if (keystorePropertiesFile.exists()) {
        keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
    }

    signingConfigs {
        release {
            if (keystoreProperties['KEYSTORE_FILE']) {
                storeFile file(keystoreProperties['KEYSTORE_FILE'])
                storePassword keystoreProperties['KEYSTORE_PASSWORD']
                keyAlias keystoreProperties['KEY_ALIAS']
                keyPassword keystoreProperties['KEY_PASSWORD']
            }
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile("proguard-android.txt"), "proguard-rules.pro"
        }
    }
}
```

## 四、构建 APK

### 4.1 Windows 系统

```bash
# 进入 android 目录
cd android

# 构建 Release 版本
gradlew.bat assembleRelease

# 构建完成后 APK 位置：
# android/app/build/outputs/apk/release/app-release.apk
```

### 4.2 Mac/Linux 系统

```bash
# 进入 android 目录
cd android

# 构建 Release 版本
./gradlew assembleRelease

# 构建完成后 APK 位置：
# android/app/build/outputs/apk/release/app-release.apk
```

### 4.3 构建选项

```bash
# 构建 Debug 版本（用于测试）
gradlew.bat assembleDebug

# 清理构建缓存
gradlew.bat clean

# 重新构建
gradlew.bat clean assembleRelease

# 并行构建（更快）
gradlew.bat assembleRelease --parallel

# 查看构建详情
gradlew.bat assembleRelease --info
```

### 4.4 构建 AAB（用于 Google Play）

```bash
# 构建 Android App Bundle
gradlew.bat bundleRelease

# 构建完成后 AAB 位置：
# android/app/build/outputs/bundle/release/app-release.aab
```

### 4.5 一键构建命令

```bash
# Windows 一键构建
npx expo prebuild --platform android && cd android && gradlew.bat assembleRelease

# Mac/Linux 一键构建
npx expo prebuild --platform android && cd android && ./gradlew assembleRelease
```

### 4.6 优化构建速度

编辑 `android/gradle.properties`，增加内存配置：

```properties
org.gradle.jvmargs=-Xmx4096m -XX:MaxMetaspaceSize=512m
org.gradle.parallel=true
org.gradle.caching=true
org.gradle.daemon=true
```

## 六、常见问题

### Q1：构建失败，提示 "SDK location not found"

**解决方案：**

检查 `local.properties` 文件：
```bash
# 创建或编辑 android/local.properties
sdk.dir=C\:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk
```

或设置环境变量：
```bash
# Windows
set ANDROID_HOME=C:\Users\你的用户名\AppData\Local\Android\Sdk

# 添加到 PATH
set PATH=%PATH%;%ANDROID_HOME%\platform-tools
```

---

### Q2：证书路径错误

**解决方案：**

1. 确认证书路径使用正斜杠 `/` 而非反斜杠 `\`
2. 检查 `keystore.properties` 中的路径是否正确：
   ```properties
   KEYSTORE_FILE=D:/keys/tentech.keystore  # 正确
   KEYSTORE_FILE=D:\keys\tentech.keystore  # 错误
   ```

---

### Q3：Gradle 构建超时

**解决方案：**

```bash
# 增加超时时间
# 编辑 android/gradle.properties
org.gradle.daemon=true
org.gradle.jvmargs=-Xmx4096m -XX:MaxHeapFreeRatio=75 -XX:MinHeapFreeRatio=25

# 使用镜像源（中国用户）
# 编辑 android/build.gradle
repositories {
    maven { url 'https://maven.aliyun.com/repository/google' }
    maven { url 'https://maven.aliyun.com/repository/jcenter' }
    maven { url 'https://maven.aliyun.com/repository/public' }
}
```

---

### Q4：构建失败，提示 "Failed to resolve dependencies"

**解决方案：**

```bash
# 清理 Gradle 缓存
cd android
rd /s /q .gradle
rd /s /q app\build

# 重新构建
gradlew.bat clean
.\gradlew.bat assembleRelease
```

---

### Q5：APK 安装失败 "INSTALL_FAILED_UPDATE_INCOMPATIBLE"

**解决方案：**

```bash
# 卸载旧版本
adb uninstall com.anonymous.bitcoinwallet

# 重新安装
adb install app-release.apk
```

## 八、高级配置

### 8.1 自定义应用图标

编辑 `app.json`：
```json
{
  "expo": {
    "icon": "./assets/images/icon.png",
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/images/android-icon-foreground.png",
        "backgroundImage": "./assets/images/android-icon-background.png",
        "backgroundColor": "#E6F4FE"
      }
    }
  }
}
```

修改后重新运行 `npx expo prebuild --platform android`。

### 8.2 修改应用名称和版本

编辑 `app.json`：
```json
{
  "expo": {
    "name": "Bitcoin Wallet",
    "version": "1.0.1",
    "android": {
      "package": "com.bitcoin.wallet"
    }
  }
}
```

修改后重新运行 `npx expo prebuild --platform android`。

### 8.3 配置应用权限

编辑 `app.json`：
```json
{
  "expo": {
    "android": {
      "permissions": [
        "INTERNET",
        "ACCESS_NETWORK_STATE",
        "CAMERA",
        "READ_EXTERNAL_STORAGE",
        "WRITE_EXTERNAL_STORAGE"
      ]
    }
  }
}
```

## 九、与 EAS 构建的区别

| 特性 | 本地构建 | EAS 云构建 |
|------|---------|-----------|
| 环境要求 | 需要 Android SDK | 不需要 |
| 构建速度 | 取决于本地配置 | 通常更快 |
| 签名管理 | 本地证书 | 自动管理 |
| 成本 | 免费 | 有免费额度 |
| 便利性 | 需要维护环境 | 一键构建 |
| 适用场景 | 开发调试、定制化 | 正式发布 |
